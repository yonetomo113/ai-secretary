# AI Secretary

**Claude Code で動く、あなた専用の AI 秘書テンプレートです。**

毎朝のスケジュール確認、メールの優先度整理、会議準備、リサーチ、タスク管理など、
日々の業務をAIがサポートします。初回起動時に対話形式であなたの情報を設定するので、
技術的な知識がなくてもすぐに使い始められます。

## 1. はじめる前に必要なもの

### 必須

- **Claude Code**（Anthropic の AI アシスタントツール）
  - まだインストールしていない場合は、以下のページを参照してください
  - https://docs.anthropic.com/en/docs/claude-code/overview
  - Mac / Windows / Linux に対応しています

### あると便利（なくても使えます）

AI秘書がメールやカレンダーに直接アクセスするには、追加のツール連携が必要です。
連携しない場合でも、あなたがメールやスケジュールをコピペすれば AI が処理します。

| 連携したいもの | 必要なツール（例） |
|--------------|------------------|
| Gmail | Gmail MCP サーバー |
| Google Calendar | Google Calendar MCP サーバー |
| Slack | Slack MCP サーバー |
| Notion | Notion MCP サーバー |
| GitHub | GitHub CLI（`gh`） |

MCP サーバーの設定がわからない場合は、後から設定できますので、
まずはツール連携なしで始めてください。

## 2. インストール手順

### 方法A: GitHub からダウンロード（推奨）

1. このリポジトリのページで「**Use this template**」ボタンを押す
   - または「**Code**」→「**Download ZIP**」でダウンロード
2. ダウンロードしたフォルダを、パソコン上の好きな場所に置く

### 方法B: ターミナルからクローン

ターミナル（コマンドライン）を使える方は、以下のコマンドを実行してください。

```bash
git clone https://github.com/YOUR_NAME/ai-secretary.git
cd ai-secretary
```

### インストール後のフォルダ構成

```
ai-secretary/
├── README.md          ← このファイル（ガイド）
├── CLAUDE.md          ← AI秘書の行動ルール
├── config/            ← あなたの設定
├── protocols/         ← 業務マニュアル
├── memory/            ← AIの学習記録
├── projects/          ← 案件管理
├── areas/             ← 継続管理する情報
├── resources/         ← 参考情報・テンプレート
└── archive/           ← 完了した案件
```

## 3. 初回セットアップ（自動）

### AI秘書を起動する

ターミナルで `ai-secretary` フォルダに移動し、Claude Code を起動します。

```bash
cd ai-secretary
claude
```

Claude Code のデスクトップアプリやVS Code拡張を使っている場合は、
`ai-secretary` フォルダを開いてから Claude を起動してください。

### 対話形式のセットアップが始まります

初回起動時、AI秘書が自動であなたに質問します。
すべてチャットで答えるだけで完了します。

```
AI秘書: 「AI秘書のセットアップを始めます。いくつか質問させてください。」

AI秘書: 「お名前（呼び方）を教えてください」
あなた: 「田中です」

AI秘書: 「役職・お立場を教えてください」
あなた: 「マーケティング部長です」

AI秘書: 「主な責務を2〜3つ教えてください」
あなた: 「マーケ戦略の立案、チームマネジメント、予算管理」

AI秘書: 「報告の好みを教えてください: 簡潔 / 詳細 / 要点+根拠」
あなた: 「簡潔で」

AI秘書: 「普段使っているツールを教えてください（複数可）」
あなた: 「GmailとGoogleカレンダーとSlack」

AI秘書: 「セットアップが完了しました。何から始めますか？」
```

所要時間は約3〜5分です。
答えたくない質問はスキップしても構いません。後から設定を変更できます。

### セットアップ後に保存されるもの

あなたの回答は以下のファイルに保存されます:

- `config/profile.md` — 名前、役職、仕事スタイルなど
- `config/tools.md` — 使用ツールの設定

これらのファイルは、あなたのパソコン上にのみ存在します。
外部に送信されることはありません。

## 4. 毎日の使い方

### 朝: 1日のスタート

Claude Code を起動し、「**おはよう**」と話しかけてください。
AI秘書がスケジュール・メール・タスクをまとめたブリーフィングを作成します。
（詳しくは [5-1. 朝ブリーフィング](#5-1-朝ブリーフィング) を参照）

### 日中: 必要なときに話しかける

何かやってほしいことがあれば、普通の言葉で話しかけるだけです。

```
「メール確認して」       → メールを優先度で分類してくれる
「14時の会議の準備して」  → 会議に必要な情報をまとめてくれる
「議事録を作って」       → 会議内容を構造化してくれる
「〜について調べて」     → リサーチレポートを作成してくれる
「〜をお願い」          → プロジェクトとして登録・管理してくれる
```

各機能の詳しい使い方は [5. 機能ガイド](#5-機能ガイド) で説明しています。

### 夕方: セッションを終える

「**ありがとう**」「**終わり**」などと言ってセッションを終了すると、
AI秘書が今日の対話から学んだことを自動で記録します。
次回のセッションでは、この学習が自動的に反映されます。
（詳しくは [5-6. AI学習・記憶](#5-6-ai学習記憶) を参照）

## 5. 機能ガイド

### 5-1. 朝ブリーフィング

**「おはよう」「今日の予定は？」「ブリーフィング」で起動します。**

今日1日の全体像を30秒で把握できるレポートを作成する機能です。

#### 話しかけ方の例

```
あなた: 「おはよう」
あなた: 「今日の予定を教えて」
あなた: 「今日のブリーフィングをお願い」
```

#### AI秘書がやること

1. カレンダーから今日の予定を取得します
2. 未読メールを優先度で分類します
3. 進行中プロジェクトの期限を確認します
4. 以下の形式でレポートを出力します

#### 出力例

```
## 今日のポイント
- 15時のA社ミーティングで予算承認の判断が必要
- B社からの見積もり依頼に本日中に返信が必要

## スケジュール
- 10:00 チーム朝会（準備: 先週の数値レポートを確認）
- 13:00 ランチミーティング 佐藤さん
- 15:00 A社 定例ミーティング（準備: 予算案の最終確認）

## メール（要対応）
- 「見積もりの件」from B社 山田様 — 3月分の見積もり依頼。本日中に返信推奨
- 「来月のイベントについて」from 社内 鈴木 — 会場候補3件の比較。判断待ち

## タスク・締切
- PRJ-003 提案書作成: 納期 4/5（残り6日）
- PRJ-001 Webサイトリニューアル: 次のマイルストーン 4/1

## 判断待ち
- B社見積もり: 標準プランで回答してよいか？
```

#### ツール連携がない場合

カレンダーやメールのツール連携をしていなくても使えます。
AI秘書が「予定を貼り付けてもらえますか？」と聞くので、
カレンダーアプリやメール画面からコピペするだけでOKです。

```
あなた: 「おはよう」
AI秘書: 「カレンダーが未接続です。今日の予定を貼り付けてもらえますか？」
あなた: 「10時 朝会、13時 ランチ 佐藤さん、15時 A社ミーティング」
AI秘書: （ブリーフィングを作成）
```

#### 保存先

ブリーフィングは `areas/daily-briefings/2026-03-30.md` のように日付付きで自動保存されます。
過去のブリーフィングを振り返りたいときは、このフォルダを確認してください。

### 5-2. メールトリアージ

**「メール確認して」「受信トリアージ」で起動します。**

未読メールを優先度で自動分類し、返信案まで作成する機能です。

#### 話しかけ方の例

```
あなた: 「メール確認して」
あなた: 「今日届いたメールを整理して」
あなた: 「受信トリアージお願い」
```

#### ツール連携がない場合の使い方

メール画面を開いて、未読メールの一覧をコピペしてください。

```
あなた: 「以下のメールを整理して」

（以下、メール画面からコピペ）
1. 件名: 見積もりの件 / 送信者: B社 山田 / 3月30日 9:15
   本文: 先日ご相談した件について、見積もりをお送りいただけますか...
2. 件名: 【週報】マーケチーム / 送信者: 鈴木 / 3月30日 8:30
   本文: 今週の実績サマリーです...
3. 件名: セミナーのご案内 / 送信者: 株式会社XX / 3月29日 18:00
   本文: AI活用セミナーのご案内...
```

#### AI秘書がやること

メールを4段階の優先度に分類します:

| 優先度 | どんなメール？ | AIの対応 |
|--------|-------------|---------|
| **高** | 返信期限あり、お金の話、クレーム、上司からの依頼 | 本文の要約 + 返信案を作成 |
| **中** | 返信は必要だが急ぎではない | 1行要約 + 推奨アクションを提示 |
| **低** | CC、ニュースレター、お知らせ | 1行要約のみ。対応不要 |
| **無視** | スパム、広告 | 件数だけ報告 |

#### 出力例

```
## メールトリアージ（2026-03-30）

### 高（即対応: 1件）
1. 「見積もりの件」from B社 山田様（9:15）
   - 要点: 先日の打ち合わせの件で見積もり依頼。今週中の回答を希望
   - 推奨: 標準プランの見積もりを本日中に送付
   - 返信案:
     山田様
     お世話になっております。
     見積もりの件、承知いたしました。本日中にお送りいたします。

### 中（当日対応: 1件）
1. 「【週報】マーケチーム」from 鈴木
   - 要点: 今週のKPI達成率85%。SNS施策が好調
   - 推奨: 確認済みの返信。SNS施策の継続を承認

### 低（確認のみ: 1件）
- 「セミナーのご案内」 — AI活用セミナー（4/15開催）。興味があれば申込

### 無視（0件除外）
```

#### 返信案について

優先度「高」のメールには、AI秘書が返信の下書きを自動で作成します。
そのまま使っても、修正を加えても構いません。

```
あなた: 「返信案の2番目、もう少し丁寧な表現にして」
AI秘書: （修正した返信案を提示）
```

#### 保存先

トリアージ結果は `areas/email-logs/2026-03-30.md` に自動保存されます。

### 5-3. 会議準備・議事録

**「会議準備」「議事録」「ミーティング」で起動します。**

会議前の準備メモ作成と、会議後の議事録作成の2つの機能があります。

#### 会議前: 準備メモを作る

```
あなた: 「15時のA社ミーティングの準備して」
あなた: 「明日の定例の準備をお願い」
```

AI秘書が以下をまとめた準備メモを作成します:
- 参加者と役割
- 議題の一覧
- 前回の議事録からの持ち越し事項（過去の議事録がある場合）
- 準備すべき資料・データ

##### 出力例（会議準備）

```
## A社 定例ミーティング（2026-03-30 15:00）

### 参加者
- 田中（自社・マーケ部長）
- A社 佐々木様（営業部長）
- A社 中村様（マーケ担当）

### 議題
- Q1実績の共有
- Q2のキャンペーン予算の承認
- 新サービスのプロモーション計画

### 前回からの持ち越し
- SNSキャンペーンの効果測定結果 → 田中が共有予定
- 新サービスのLP案 → A社 中村様が持参予定

### 準備事項
- Q1実績の数値レポート（売上・CVR・CPA）
- Q2予算案（3パターン）
- SNSキャンペーンの効果測定データ
```

##### ツール連携がない場合

```
あなた: 「15時の会議の準備して」
AI秘書: 「カレンダーが未接続です。会議の情報を教えてください
        （参加者、議題、前回の会議内容など）」
あなた: 「A社の佐々木さんと中村さん。議題はQ1実績とQ2予算の話」
AI秘書: （準備メモを作成）
```

#### 会議後: 議事録を作る

会議が終わったら、メモや書き起こしをAI秘書に渡して議事録を作成します。

```
あなた: 「この会議の議事録を作って」

（以下、会議メモをコピペ）
A社ミーティング
- Q1は目標の92%達成。SNSが好調で前年比120%
- Q2予算はプランBの500万で承認
- 新サービスLPは4/10までに初稿、4/20にレビュー
- 佐々木さんが社内稟議を4/3までに通す
- 次回は4/14 15時
```

##### 出力例（議事録）

```
## 議事録: A社 定例ミーティング（2026-03-30）

### 参加者
- 田中（自社）
- 佐々木様（A社）
- 中村様（A社）

### 決定事項
- Q2キャンペーン予算: プランB（500万円）で承認
- 新サービスLP初稿の締切: 4/10
- LPレビュー日程: 4/20

### アクションアイテム
| 担当 | タスク | 期限 |
|------|--------|------|
| 田中 | LP初稿を作成 | 4/10 |
| 佐々木様 | 社内稟議を通す | 4/3 |
| 中村様 | LP素材の準備 | 4/8 |

### 議論メモ
- Q1実績は目標の92%。SNS施策が前年比120%と好調
- Q2はSNS施策を継続しつつ、新サービスのプロモーションを追加

### 次回予定
- 4/14（月）15:00
```

議事録の中にあなたのタスク（上の例では「LP初稿を作成」）があれば、
AI秘書がプロジェクトへの登録を提案します。

#### 保存先

- 準備メモ・議事録ともに `areas/meeting-notes/2026-03-30-A社定例.md` に保存されます
- 同じ会議の準備メモと議事録は1つのファイルにまとまります

### 5-4. リサーチ

**「調べて」「リサーチ」「競合」で起動します。**

Webやドキュメントを調査し、ソース付きの構造化レポートを作成する機能です。

#### 話しかけ方の例

```
あなた: 「競合のA社について調べて」
あなた: 「最近のSNSマーケティングのトレンドをリサーチして」
あなた: 「リモートワークツールを比較調査して」
あなた: 「来月のイベント会場の候補を3つ調べて」
```

#### AI秘書がやること

1. まず「何を・なぜ・どこまで調べるか」を1文で宣言します
2. Web検索やドキュメント調査を実行します
3. 発見を整理し、構造化レポートを作成します
4. 最後に「だから何をすべきか」のアクション案を提示します

#### 出力例

```
あなた: 「リモートワークツールを比較調査して」
```

```
## リサーチ: リモートワークツール比較（2026-03-30）

### 調査スコープ
チーム（10〜30名規模）向けのリモートワークツールを比較。
ビデオ会議・チャット・タスク管理の3軸で評価する。

### 要約（3行以内）
コスパ重視ならGoogle Workspace、機能統合重視ならMicrosoft 365、
チャット中心のチームならSlack + Zoom の組み合わせが最適。

### 詳細

#### ビデオ会議
| ツール | 月額/人 | 最大参加者 | 録画 | 特徴 |
|--------|--------|----------|------|------|
| Google Meet | ¥680〜 | 100人 | 有 | Workspace統合 |
| Zoom | ¥1,370〜 | 300人 | 有 | 高品質・安定性 |
| Teams | ¥750〜 | 300人 | 有 | Office統合 |

#### チャット
- Slack: チャンネル整理が優秀。外部連携が豊富
- Teams: Office統合。社内利用には十分
- Google Chat: シンプル。Workspace利用者には追加コストなし

#### タスク管理
- Notion: 柔軟性が高い。学習コストがやや高め
- Asana: タスク管理に特化。直感的なUI
- Linear: 開発チーム向け。高速なUX

### アクション案
1. Google Workspace を導入（理由: 既にGmail利用中、追加コスト最小）
2. Slack + Zoom の組み合わせ（理由: チャット品質重視の場合）

### ソース
- Google Workspace 料金ページ: https://workspace.google.com/pricing
- Zoom 料金ページ: https://zoom.us/pricing
- ...
```

#### 品質について

AI秘書のリサーチには以下のルールがあります:
- **事実と推測を分ける**: 推測には「（推測）」と明記します
- **ソースを必ず記載**: 根拠のない情報は含めません
- **比較は表形式**: 複数のものを比較する場合、見やすい表で整理します

#### 深掘りしたいとき

```
あなた: 「Google Workspace についてもう少し詳しく調べて」
あなた: 「セキュリティ面での比較も追加して」
あなた: 「このリサーチをもとに提案書を作って」
```

追加の調査を依頼したり、リサーチ結果をもとに別のドキュメント作成を依頼できます。

#### 保存先

- プロジェクトに関連するリサーチ → `projects/PRJ-XXX/resources/` に保存
- 一般的なリサーチ → `resources/industry/2026-03-30-リモートワークツール比較.md` に保存

### 5-5. プロジェクト管理

**新しい仕事の依頼をすると自動で起動します。**

あなたが何か仕事を依頼すると、AI秘書が自動的にプロジェクトとして登録し、
成果物やメモをフォルダで整理して管理する機能です。

#### 新しいプロジェクトを始める

普通に仕事を依頼するだけで、AI秘書が自動的にプロジェクトを作成します。

```
あなた: 「来月のセミナー用のプレゼン資料を作って」
```

AI秘書が裏側で以下を自動実行します:
1. `projects/INDEX.md`（プロジェクト一覧）を確認
2. 新しいプロジェクト番号（PRJ-001）を採番
3. `projects/PRJ-001_セミナープレゼン/` フォルダを作成
4. README.md（依頼内容・ステータス・進捗）を作成
5. 作業を開始

作成されるフォルダ構成:

```
projects/PRJ-001_セミナープレゼン/
├── README.md       ← 依頼内容「セミナー用プレゼン作成」、ステータス「進行中」
├── output/         ← 成果物（プレゼンファイル等）がここに入る
├── resources/      ← 参考資料がここに入る
├── notes/          ← 作業メモがここに入る
```

#### プロジェクト一覧を確認する

```
あなた: 「今のプロジェクト一覧を見せて」
あなた: 「進行中の案件は？」
```

AI秘書が `projects/INDEX.md` をもとに一覧を提示します:

```
## プロジェクト一覧

| 番号 | タイトル | ステータス | 納期 |
|------|---------|----------|------|
| PRJ-001 | セミナープレゼン | 進行中 | 4/15 |
| PRJ-002 | Webサイトリニューアル | 進行中 | 5/1 |
| PRJ-003 | 競合調査レポート | 未着手 | 4/10 |
```

#### 既存プロジェクトに追加作業を依頼する

同じ案件への追加依頼は、AI秘書が自動で同じプロジェクトに紐づけます。

```
あなた: 「セミナープレゼンに、Q1の実績データも追加して」
```

AI秘書は既存の `PRJ-001_セミナープレゼン/` に追加作業として対応します。
新しいフォルダは作られません。

#### 成果物のバージョン管理

修正版が出た場合、バージョンを付けて管理されます。

```
projects/PRJ-001_セミナープレゼン/
└── output/
    ├── v1_セミナープレゼン.md    ← 初稿
    └── v2_セミナープレゼン.md    ← 修正版
```

#### プロジェクトを完了する

```
あなた: 「セミナープレゼンの件は完了で」
あなた: 「PRJ-001を完了にして」
```

AI秘書が以下を実行します:
1. README.md のステータスを「完了」に更新
2. INDEX.md のステータスを「完了」に更新
3. プロジェクトフォルダを `archive/` に移動

```
完了前: projects/PRJ-001_セミナープレゼン/
完了後: archive/PRJ-001_セミナープレゼン/
```

完了した案件は `archive/` に保管されるので、後から振り返ることも可能です。

#### プロジェクトの進捗を確認する

```
あなた: 「PRJ-002の進捗を教えて」
あなた: 「Webサイトリニューアルの状況は？」
```

AI秘書がプロジェクトの README.md や成果物を確認し、現状を報告します。

### 5-6. AI学習・記憶

**自動で動きます。特別な操作は不要です。**

AI秘書はあなたとのやりとりから自動的に学習し、次回以降の対応に活かします。

#### セッション終了時の自動記録

「ありがとう」「終わり」などでセッションを終了すると、AI秘書が自動で以下を記録します:

- **修正の記録**: やり直しが発生した場合、その原因と正解パターン
- **発見の記録**: ツール操作で新しくわかった注意点
- **改善の記録**: 次回から変えるべき対応方法

例（`memory/learnings.md` に自動追記される内容）:

```
### 2026-03-30

- **修正**: メールの返信案がフォーマルすぎた。カジュアルな文体を好む
- **発見**: カレンダーAPIの日時指定でタイムゾーンを明示する必要がある
- **改善**: リサーチレポートは表を多用した方が読みやすいとのフィードバック
```

#### 積極的学習（常時）

セッション中も、AI秘書は以下を自動検出して `memory/patterns.md` に記録します:

| 検出するもの | 例 |
|-------------|-----|
| 繰り返し使う指示パターン | 毎朝「おはよう」でブリーフィングを依頼している |
| 修正フィードバック | 「もっと短く」「表にして」「具体例を入れて」 |
| 好みの傾向 | 箇条書きを好む、数字には必ずグラフを求める |
| よく使う用語 | 「MTG」=ミーティング、「確認よろ」=確認お願いします |

#### 学習が反映されるタイミング

セッション開始時に、AI秘書は `memory/` フォルダの内容を読み込みます。
過去の学習が自動的に反映されるため、**使うほど精度が上がります**。

例:
- 初回: レポートを詳細に書いたら「もっと短く」と言われた
- 2回目以降: 最初から簡潔なレポートを出すようになる

#### 学習内容を確認する

```
あなた: 「今までに学習したことを見せて」
あなた: 「私の好みパターンを確認したい」
```

AI秘書が `memory/learnings.md` と `memory/patterns.md` の内容を要約して見せてくれます。

#### 学習をリセットする

学習内容をゼロに戻したい場合は、以下のファイルの中身を削除してください:
- `memory/learnings.md`
- `memory/patterns.md`

または、AI秘書に「学習をリセットして」と伝えることもできます。

## 6. フォルダの中身について

AI秘書が作成するファイルは、自動的に適切なフォルダに整理されます。
あなたが手動でファイルを動かす必要はありません。

### PARAメソッドとは？

このAI秘書は **PARA（パラ）メソッド** という情報整理術をベースにしています。
Tiago Forte氏が考案した手法で、すべての情報を4つのカテゴリに分けるシンプルなルールです。

| カテゴリ | 英語 | 意味 | このAI秘書では |
|---------|------|------|--------------|
| **P** | Projects | 期限とゴールがある仕事 | `projects/` フォルダ |
| **A** | Areas | 終わりがなく継続的に管理するもの | `areas/` フォルダ |
| **R** | Resources | いつか役立つ参考情報 | `resources/` フォルダ |
| **A** | Archive | 完了した・使わなくなったもの | `archive/` フォルダ |

#### 具体例で理解する

「プレゼン資料を4/15までに作る」は **期限とゴールがある** → **Projects**
「毎朝のブリーフィング記録」は **終わりがなく継続的** → **Areas**
「業界のリサーチ結果」は **いつか参考になる情報** → **Resources**
「完了した過去の案件」は **もう使わないが保管したい** → **Archive**

この4つの箱に情報を分けるだけで、「あのファイルどこだっけ？」がなくなります。
AI秘書が自動的にファイルを正しいフォルダに振り分けるので、
あなたはPARAのルールを覚える必要はありません。

### 全体の構成

```
ai-secretary/
│
├── config/             あなたの個人設定
│   ├── profile.md        名前・役職・仕事の好み
│   └── tools.md          使用ツールの一覧
│
├── protocols/          AI秘書の業務マニュアル
│   ├── onboarding.md     初回セットアップの手順
│   ├── daily-briefing.md 朝ブリーフィングの手順
│   ├── email-triage.md   メール分類のルール
│   ├── meeting-prep.md   会議準備・議事録の手順
│   ├── research.md       リサーチの進め方
│   └── project-mgmt.md   プロジェクト管理のルール
│
├── memory/             AIの学習記録（自動で溜まる）
│   ├── learnings.md      セッション毎の気づき
│   └── patterns.md       あなたの好み・行動パターン
│
├── projects/           進行中の案件
│   └── INDEX.md          全プロジェクトの一覧表
│
├── areas/              継続的に管理する情報（自動作成）
│   ├── daily-briefings/  朝ブリーフィングの記録
│   ├── email-logs/       メール整理の記録
│   ├── meeting-notes/    議事録
│   └── clients/          顧客・パートナー情報
│
├── resources/          参考情報の保管場所
│   └── templates/        テンプレート集
│
└── archive/            完了した案件の保管場所
```

### フォルダの役割（かんたん説明）

| フォルダ | PARA | 何が入る？ | たとえるなら |
|---------|------|----------|------------|
| `config/` | — | あなたの設定 | 秘書に渡す名刺・好みメモ |
| `protocols/` | — | 業務マニュアル | 秘書の仕事の手順書 |
| `memory/` | — | AIの学習記録 | 秘書のノート |
| `projects/` | **P** | 進行中の仕事 | デスク上の書類トレー |
| `areas/` | **A** | 日々の記録 | 日報・議事録ファイル |
| `resources/` | **R** | 参考資料 | 本棚 |
| `archive/` | **A** | 終わった仕事 | キャビネット |

### ファイルが保存される場所の早見表

| 何を作った？ | 保存先 |
|------------|--------|
| 朝ブリーフィング | `areas/daily-briefings/YYYY-MM-DD.md` |
| メールトリアージ結果 | `areas/email-logs/YYYY-MM-DD.md` |
| 議事録・会議準備 | `areas/meeting-notes/YYYY-MM-DD-会議名.md` |
| プロジェクトの成果物 | `projects/PRJ-XXX/output/` |
| プロジェクトの参考資料 | `projects/PRJ-XXX/resources/` |
| 汎用リサーチ結果 | `resources/industry/YYYY-MM-DD-テーマ.md` |
| AI学習記録 | `memory/learnings.md`, `memory/patterns.md` |

## 7. カスタマイズ方法

### 自分のプロフィールを変更したい

`config/profile.md` をテキストエディタで開いて直接編集できます。
または、AI秘書に「プロフィールを更新して」と話しかけてください。

### 使用ツールを追加・変更したい

`config/tools.md` を開き、使いたいツールの「**状態**」を「**無効**」から「**有効**」に変更します。

```
変更前: - **状態**: 無効
変更後: - **状態**: 有効
```

### AI秘書の行動を変えたい

`protocols/` フォルダの各ファイルが、AI秘書の業務マニュアルです。
テキストエディタで開いて、日本語で書き換えるだけです。

| 変えたいこと | 編集するファイル |
|-------------|----------------|
| メールの優先度基準 | `protocols/email-triage.md` |
| 朝ブリーフィングの形式 | `protocols/daily-briefing.md` |
| 議事録のフォーマット | `protocols/meeting-prep.md` |
| リサーチの進め方 | `protocols/research.md` |
| プロジェクト管理のルール | `protocols/project-mgmt.md` |
| セットアップの質問項目 | `protocols/onboarding.md` |

### AIの学習をリセットしたい

`memory/learnings.md` と `memory/patterns.md` の中身を削除してください。
AI秘書がゼロから学び直します。

## 8. よくある質問

### Q: プログラミングの知識は必要ですか？

不要です。AI秘書とのやりとりはすべて普通の日本語で行います。
ファイルの編集も、メモ帳やテキストエディタで日本語を書き換えるだけです。

### Q: 自分のデータは外部に送信されますか？

`config/` や `memory/` のファイルはあなたのパソコン上にのみ存在します。
ただし、Claude Code を使う際に会話内容は Anthropic のサーバーに送信されます。
機密性の高い情報の取り扱いについては、Anthropic のプライバシーポリシーを確認してください。

### Q: メールやカレンダーを連携しないと使えませんか？

使えます。ツール連携がなくても、AI秘書に情報をコピペで渡せば処理できます。
たとえば、メール画面の内容をコピーして「これを整理して」と渡すだけでOKです。

### Q: 途中でセットアップをやり直したい場合は？

`config/profile.md` を開いて、「名前」の欄を「（未記入）」に戻してください。
次回起動時にセットアップが最初からやり直されます。

### Q: 複数人で同じテンプレートを使えますか？

使えます。このリポジトリをテンプレートとして各自がコピーしてください。
`config/` と `memory/` は個人ごとに異なる内容になるため、
共有リポジトリとしてではなく、個人ごとにコピーして使う形を推奨します。

### Q: カスタマイズしたらAI秘書が動かなくなりました

`CLAUDE.md` は AI秘書の動作の根幹です。このファイルを大幅に変更すると
正しく動作しなくなる場合があります。`CLAUDE.md` は基本的にそのままにして、
カスタマイズは `config/` と `protocols/` で行うことを推奨します。

### Q: 「おはよう」以外の言葉でも動きますか？

はい。AI秘書は意味を理解するので、完全に同じ言葉でなくても大丈夫です。
たとえば「おはよう」の代わりに「今日の予定は？」「ブリーフィングお願い」でも
同じ朝ブリーフィング機能が動きます。

### Q: 過去のブリーフィングや議事録を見返したい

`areas/` フォルダに日付付きで自動保存されています。
AI秘書に「先週のブリーフィングを見せて」「3/25の議事録を確認したい」と
話しかけるか、フォルダを直接開いて確認できます。

### Q: プロジェクトが増えすぎたらどうなりますか？

完了したプロジェクトは `archive/` フォルダに移動されるので、
`projects/` フォルダには進行中の案件だけが残ります。
過去の案件は `archive/` でいつでも振り返れます。

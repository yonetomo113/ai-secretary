# メールトリアージ

未読メールを分類し、対応案を提示する。

## 優先度マトリクス

| 優先度 | 条件 | 対応 |
|--------|------|------|
| **高** | 返信期限あり / 金銭関連 / クレーム / 上位者からの依頼 | 即対応。本文と返信案を提示 |
| **中** | 要返信だが急ぎでない / 情報提供の依頼 | 当日中に対応。要約と推奨アクションを提示 |
| **低** | CC / ニュースレター / 通知系 | 1行要約のみ。対応不要と明記 |
| **無視** | スパム / 広告 / 自動通知で不要なもの | リストから除外。件数のみ報告 |

## 出力フォーマット

```
## メールトリアージ（YYYY-MM-DD）

### 高（即対応: N件）
1. **[件名]** from [送信者]（[受信時刻]）
   - 要点: [1〜2行]
   - 推奨: [具体的なアクション]
   - 返信案: [下書き]

### 中（当日対応: N件）
1. **[件名]** from [送信者]
   - 要点: [1行]
   - 推奨: [アクション]

### 低（確認のみ: N件）
- [件名] — [1行要約]

### 無視（N件除外）
```

## 判断に迷う場合

- 送信者の重要度が不明 → config/profile.md の責務範囲から推測する
- それでも判断できない → 「中」に分類し、ユーザーに確認を促す

## 保存

`areas/email-logs/YYYY-MM-DD.md` に保存する。

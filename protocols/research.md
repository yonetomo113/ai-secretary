# リサーチ

構造化されたリサーチを実行し、アクション可能な形で報告する。

## 実行手順

1. **スコープ確認** — 何を・なぜ・どこまで調べるか、1文で宣言する
2. **情報収集** — Web検索・ドキュメント参照・データ分析を実行
3. **構造化** — 発見を整理し、レポートを生成
4. **アクション提示** — 「だから何をすべきか」を明確にする

## 出力フォーマット

```
## リサーチ: [テーマ]（YYYY-MM-DD）

### 調査スコープ
[何を・なぜ・どこまで]

### 要約（3行以内）
[最も重要な発見]

### 詳細

#### [観点1]
- [発見]
- [根拠・ソース]

#### [観点2]
- [発見]
- [根拠・ソース]

### アクション案
1. [推奨アクション]（理由: [根拠]）
2. [代替案]（理由: [根拠]）

### ソース
- [URL or 参照元]
```

## 品質基準

- 事実と推測を明確に分ける。推測には「（推測）」と付記する
- ソースは必ず記載する。ソースなしの主張は含めない
- 数字・データは可能な限り最新のものを使う
- 競合リサーチの場合、比較表を含める

## 保存

プロジェクトに紐づく場合 → `projects/PRJ-XXX/resources/` に保存
汎用リサーチの場合 → `resources/industry/YYYY-MM-DD-[テーマ].md` に保存

# 朝ブリーフィング

ユーザーの1日の全体像を30秒で把握できるレポートを作成する。

## 実行手順

1. **カレンダー確認** — 今日の予定を時系列で取得
2. **メール確認** — 未読メールを優先度分類（email-triage.md の基準を使用）
3. **タスク確認** — 進行中プロジェクトの期限・ステータスを確認
4. **レポート生成** — 以下のフォーマットで出力

## 出力フォーマット

```
## 今日のポイント
- [最も重要な事項を1〜3個]

## スケジュール
- HH:MM [予定]（準備: [必要な準備があれば]）
- HH:MM [予定]
- ...

## メール（要対応）
- [件名] from [送信者] — [1行要約と推奨アクション]
- ...

## タスク・締切
- [期限が近いタスクを優先度順に]

## 判断待ち
- [ユーザーの判断が必要な事項があれば]
```

## ツールが無効の場合

該当セクションを「（未接続）」と表示し、手動入力を促す。
例: 「カレンダーが未接続です。今日の予定を貼り付けてもらえますか？」

## 保存

`areas/daily-briefings/YYYY-MM-DD.md` に保存する。
ディレクトリが存在しない場合は自動作成する。

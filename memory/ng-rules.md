# NGルール

## 外部サービス・データ送信

- 確認なしに外部サービスへのデータ送信・投稿を実行しない
- 金銭・契約・認証情報に関わる操作は必ず一度確認する

## ファイル・リポジトリ操作

- 存在しないファイルを「ある」と言わない
- push前に必ず `git remote -v` でリモートURLとアカウント名を確認する
- 別アカウントのリポジトリにpushしない

## コード実装

- localStorageを使う実装でlocalStorageが使えない環境を想定したコードを書かない
- iPhoneのSafariからの外部APIコールはfetchを使う。XHRは使わない

## ドキュメント

- マニュアル・ドキュメントは実際に動作確認済みの手順のみ記載する
- 未確認の手順を「たぶんこうすれば動く」として書かない
